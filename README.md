# 低空物流服务优化：多无人机协同调度

中山大学深圳校区 · 运筹学大作业

---

## 1. 项目背景

中山大学深圳校区快递驿站（西园快递站）每天有大量快递需配送至校内各用户点（食堂、宿舍、图书馆等）。本项目研究如何使用多架美团 M4 无人机高效完成配送任务，在满足载重、电池容量、时间窗等约束的前提下，使总配送成本最低。

这是一个带时间窗的车辆路径问题（VRPTW）的变体，特征在于：
- 无人机连续执行逐件配送，电量不够时回起降点换电
- 能耗与飞行距离和实时载重相关（空载省电、载货费电）

---

## 2. 数学模型

### 2.1 决策变量

- **x_i** ∈ {0,1}：无人机 i 是否启用
- **y_ir** ∈ {0,1}：无人机 i 是否选择方案 r

### 2.2 目标函数

```
min Z = F · Σ x_i                 启用成本（每启用一架付固定费用）
       + α · Σ E_ir · y_ir        能耗成本（飞得越远、装得越重越贵）
       + γ · Σ L_ir · y_ir        超时惩罚（晚于时间窗的惩罚）  
       + c_swap · Σ N_swap        换电惩罚（每换一次电的成本）
```

### 2.3 约束条件

| 约束 | 内容 | 处理方式 |
|------|------|----------|
| 任务覆盖 | 每件快递必须配送且仅配送一次 | 评估函数检查 |
| 电池约束 | 执行订单前判断剩余电量是否足够 | 电量不足时返回起降点换电 |
| 时间窗 | 早于 a_j 等待、晚于 b_j 记录超时 | 模拟执行时逐件检查 |
| 换电 | 换电耗时 5 min，产生换电成本 | 飞回起降点后等待 |

### 2.4 能耗模型

每段飞行的能耗根据是否载货分别计算：

**空载段（去取货的路上）：**

```
能耗 = e0 × 距离
```

**载货段（取完货去配送的路上）：**

```
能耗 = (e0 + e1 × 当前载重) × 距离
```

其中 e0 为基础能耗系数（0.1 Wh/m），e1 为载重附加系数（0.01 Wh/(m·kg)）。当前载重随配送过程逐渐减轻。

## 3. 算法

### 3.1 随机搜索

1. 按重量从大到小遍历订单
2. 将每件订单随机分配给无人机
3. 固定时间预算内不停尝试，取成本最低的方案

### 3.2 遗传算法

| 参数 | 值 |
|------|:---:|
| 种群大小 | 100 |
| 终止条件 | 固定时间预算（默认 20s）|
| 选择方式 | 锦标赛（k=5）|
| 交叉方式 | 均匀交叉（概率 0.5）|
| 变异方式 | 多点变异（概率 0.15）|
| 精英保留 | 5% |
| 种子解 | 随机搜索最优解 |

### 3.3 核心评估函数（eval_solution）

两种算法共用同一评估函数，逐件模拟配送执行：

```
输入：120 维分配向量（每件订单→某架无人机）
剩余电量 = 1000 Wh

对每架无人机：
  1. 订单按就绪时间排序，选择最近起降点
  2. 逐件执行配送：
     飞到取货点（空载，能耗 = e0 × 距离）
     飞到配送点（载货，能耗 = (e0 + e1 × 实时载重) × 距离）
     检查时间窗（早到等待，晚到累加超时）
  3. 执行下一件前判断：当前能耗+回程能耗 > 剩余电量？
     → 是：飞回起降点换电（5 min），重置电量
     → 否：继续配送

输出：总成本、总能耗、总超时、是否可行
```

## 6. 实验结果

固定时间预算 20s，每种配置重复 10 次，报告均值±标准差：

### 120 件订单

| 指标 | 随机搜索 | 遗传算法 |
|------|:---:|:---:|
| 总成本 | 37.7 ± 0.4 | **34.3 ± 0.5** |
| 启用无人机 | 3 架 | **3 架** |
| 总能耗 (Wh) | 28417 ± 506 | **25665 ± 548** |
| 超时 (min) | 1.0 ± 1.4 | **0.2 ± 0.3** |
| 换电次数 | 31 ± 1 | **27 ± 1** |

### 300 件订单

| 指标 | 随机搜索 | 遗传算法 |
|------|:---:|:---:|
| 总成本 | 99.6 ± 1.0 | **87.2 ± 1.0** |
| 启用无人机 | 7 架 | **7 架** |
| 总能耗 (Wh) | 75934 ± 1080 | **66205 ± 938** |
| 超时 (min) | 3.9 ± 4.4 | **0.0 ± 0.1** |
| 换电次数 | 86 ± 2 | **72 ± 2** |

### 小结

- 遗传算法较随机搜索成本降低约 **9~12%**，能耗降低约 **10~13%**
- 遗传算法在两个数据集上均能稳定控制在 **0 超时**附近
- 两个数据集的换电次数均符合预期，电量判断机制工作正常

详细原始数据见 `docs/experiment_results.csv`。
