# 校园驿站快递配送 - 多无人机协同调度数学模型

## 1. 问题描述

中山大学深圳校区快递驿站（起点 s）有 m 件快递需由 n 架美团 M4 无人机配送至校内各用户配送点。每件快递有重量 wj、服务时间窗 [aj, bj]；每架无人机有最大载重 Wi、电池容量 Ei、巡航速度 vi。需确定：启用哪些无人机、每架配送哪些快递、按什么顺序、何时出发返回，使总成本最低。

## 2. 集合与参数

### 2.1 集合

- I = {1, 2, ..., n}  无人机集合
- J = {1, 2, ..., m}  快递任务集合
- U = {0, 1, 2, ..., m}  节点集合，0 为驿站 s，j>=1 为配送点 cj
- Ri  无人机 i 的候选完整调度方案集合

### 2.2 已知参数

- s: 驿站位置，西园快递站 (72, -244) m
- cj: 快递 j 的用户配送点坐标
- wj: 快递 j 重量 (kg)
- [aj, bj]: 快递 j 服务时间窗 (h)
- Sj: 快递 j 可装载出发时间 (h)
- duv: 节点 u 到 v 的欧氏距离 (m)
- vi: 无人机 i 巡航速度 (m/s)，M4: 15 m/s
- Wi: 无人机 i 最大载重 (kg)，M4: 2.5 kg
- Ei: 无人机 i 电池容量 (Wh)，M4: 1000 Wh
- Fi: 启用无人机 i 的固定成本，M4: 100
- H: 订单拆分管理成本，50
- alpha: 时间成本权重，1.0
- beta: 能耗成本权重，0.3
- gamma: 超时惩罚权重，2.0
- t_prep: 起飞/装载准备时间 (h)，1 min
- t_swap: 换电/充电时间 (h)，2 min

### 2.3 决策变量

- xi: 0-1 变量，无人机 i 是否被启用
- yir: 0-1 变量，无人机 i 是否选择执行候选方案 r

## 3. 候选调度方案生成

一个候选方案 r 属于 Ri 表示无人机 i 的一次完整调度（可能包含多趟），每趟从驿站出发、配送若干快递、返回驿站。

### 3.1 单趟路线

无人机 i 的一趟配送路线为：s -> cj1 -> cj2 -> ... -> cjq -> s

其中 {j1, j2, ..., jq} 为该趟装载的快递集合。

该路线需满足：

(C1) 载重约束：初始装载重量不超过无人机最大载重

  求和(wj) <= Wi  （对所有 j 属于该趟）

(C2) 电量约束：单趟飞行总能耗不超过电池容量

  E_trip = 求和(e_seg) <= Ei

  其中每段能耗：
  - 空载段（取货途中）：e_seg = e0 * duv
  - 载货段（送货途中）：e_seg = e0 * duv + e1 * duv * load

  e0：基础能耗系数（与距离成正比），0.5
  e1：载重相关能耗系数（与距离x载重成正比），0.05
  取货段 load = 0（无人机空载飞往取货点）

(C3) 时间窗约束：每件快递的送达时间在时间窗内

  aj <= t_arrive_j <= bj  对所有 j

  - 早于 aj 到达：等待至 aj 后投递
  - 晚于 bj 到达：记录超时量 L = t_arrive_j - bj

(C4) 飞行时间约束：单趟总飞行时间不超过最大续航

  t_flight = d_total / vi <= t_max

  t_max = 11 min = 11/60 h（M4 参数）

### 3.2 多趟调度方案

若无人机 i 执行多趟配送，完整方案 r 为多条单趟路线按时间顺序组合：

  r = (trip_1, trip_2, ..., trip_K)

趟间约束：

  t_return_k + t_swap <= t_depart_{k+1}  对所有 k

每趟从驿站出发时重新装载，载重和电量独立计算。

### 3.3 方案属性预计算

对每个候选方案 r 属于 Ri，预先计算：

- A_jr：覆盖参数，方案 r 是否配送快递 j（1/0）
- T_ir：方案 r 的总配送时间 (h)，含飞行+等待
- E_ir：方案 r 的总能耗 (Wh)
- L_ir：方案 r 的总超时量 (h)
- C_ir：方案 r 的综合运行成本

综合运行成本：

  C_ir = alpha * T_ir + beta * E_ir + gamma * L_ir

### 3.4 方案生成方法

1. 最近邻启发式：每次选距离当前位最近的未服务快递
2. 枚举所有可行子集（大小 1-3），检查载重/电量/时间窗

## 4. 优化模型

### 4.1 目标函数

  min Z = 求和_i 求和_r C_ir * y_ir + 求和_i Fi * xi

### 4.2 约束条件

(C5) 快递覆盖：每件快递必须被恰好一个方案覆盖

  求和_i 求和_r A_jr * y_ir = 1,  对所有 j 属于 J

(C6) 方案选择：每架无人机最多选择一个方案，且只有启用的无人机才能选方案

  求和_r y_ir <= xi,  对所有 i 属于 I

(C7) 变量域：

  xi 属于 {0, 1},  对所有 i 属于 I
  yir 属于 {0, 1},  对所有 i 属于 I, r 属于 Ri

## 5. 完整模型汇总

目标函数：
  min Z = 求和_i=1^n 求和_r C_ir * y_ir + 求和_i=1^n Fi * xi

约束条件：
  (C5) 求和_i 求和_r A_jr * y_ir = 1,  对所有 j （快递覆盖）
  (C6) 求和_r y_ir <= xi,  对所有 i （方案选择）
  (C7) xi, yir 属于 {0,1}

其中：
  C_ir = alpha * T_ir + beta * E_ir + gamma * L_ir

载重、电量、时间窗、飞行时间约束已在候选方案生成阶段处理（不可行方案不进入 Ri）。

## 6. 算法流程

  Step 1: 输入数据（节点坐标、快递信息、无人机参数）
  Step 2: 计算距离矩阵 duv
  Step 3: 对每架无人机 i，生成候选方案集合 Ri
          - 枚举所有可行单趟路线（满足 C1-C4）
          - 组合多趟路线（满足趟间时间约束）
  Step 4: 对每个候选方案，计算 A_jr, T_ir, E_ir, L_ir, C_ir
  Step 5: 求解整数规划模型（目标 Z，约束 C5-C7）
  Step 6: 输出最优方案

## 7. 无人机参数（美团 M4）

- 最大载重 Wi: 2.5 kg
- 巡航速度 vi: 15 m/s = 54 km/h
- 电池容量 Ei: 1000 Wh
- 最大航程: 10 km
- 最大续航: 11 min
- 配送半径: 5 km
- 启用成本 Fi: 100
- 拆分惩罚 H: 50
